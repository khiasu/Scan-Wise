# Question Bank Digitizer Developer Documentation

Question Bank Digitizer is a local-first Android application built using Jetpack Compose, Room DB, WorkManager, and EncryptedSharedPreferences.

---

## Architectural Systems

```
app/src/main/java/com/khiasu/docscanai/
│
├── data/              # Database entities, DAO contracts, and shared QuestionParser
├── export/            # Exporters for DOCX, PDF, CSV, JSON, and MediaStore writer
├── network/           # API Clients (Groq, Gemini, Claude, OpenAI)
├── prefs/             # Preferences handlers (Encrypted Credentials & Theme settings)
├── ui/                # Compose screens (Scan, Library, Detail, Settings) and Navigation
└── worker/            # WorkManager task for background page text extraction
```

### 1. Database & Persistence
We use a **Room Database** defined in [AppDatabase.kt](file:///e:/QuestBank_AI/app/src/main/java/com/khiasu/docscanai/data/AppDatabase.kt) with two core tables:
- **`documents`**: Holds the parent document entity (title, creation time, source type, client details).
- **`pages`**: Holds individual page records linked to a parent document via a foreign key constraint. We index `documentId` to prevent full table scans.
- **DAO Operations**: Supports page insertion, page-level deletion (`deletePage`), and document-level cascade deletion.

### 2. Shared Parsing Engine (`QuestionParser.kt`)
The text blocks and solution keys generated by AI models are parsed systematically in [QuestionParser.kt](file:///e:/QuestBank_AI/app/src/main/java/com/khiasu/docscanai/data/QuestionParser.kt):
- **`ParsedQuestionData`**: Data class representing a structured question (type, marks, question text, paraphrased version, options, correct answer key, brief explanation, and hint).
- **`parseQuestionValue`**: A regex-backed parser that compiles unstructured text generated by LLMs into a structured `ParsedQuestionData` object.
- **`serializeQuestionValue`**: Serializes structured question data back into a readable, formatted text format.

### 3. Background Page Processing
When a new scan is created, **WorkManager** in [ProcessPageWorker.kt](file:///e:/QuestBank_AI/app/src/main/java/com/khiasu/docscanai/worker/ProcessPageWorker.kt) enqueues a background task:
- Performs initial text extraction (OCR transcription) using the selected cloud vision model.
- Employs a lightweight prompt to ensure the extraction is rapid and resilient.
- Scanned page cards display local image thumbnails on the left during extraction to keep users engaged.

### 4. Interactive Question Solver
When a teacher triggers **Solve & Generate Answers** in the Question Bank UI:
- The app sends the transcribed raw text to Gemini or the active model using `SOLVE_PROMPT` in [AiClient.kt](file:///e:/QuestBank_AI/app/src/main/java/com/khiasu/docscanai/network/AiClient.kt).
- Instructs the AI model to solve the test paper, determine the answer key, and generate a brief, one-sentence explanation and tip.
- On completion, the structured output is saved to the database (`fieldsJson`) and the solved formatted sheet is saved as the page's editable raw text.

### 5. Practice Quiz UI
Inside [DocumentDetailScreen.kt](file:///e:/QuestBank_AI/app/src/main/java/com/khiasu/docscanai/ui/DocumentDetailScreen.kt):
- Renders an interactive quiz mode that parses active questions.
- MCQ options are clickable with visual feedback (correct/incorrect answer highlights).
- Subjective questions allow user text input, rubric disclosure, and self-grading.
- Tracks performance metrics and updates a completion progress bar.

### 6. Structured Exporters
Exporters in `export/` parse question values into clean structures:
- **CSV**: Compiles fields using `QuestionParser` to write individual question details, options, answers, and hints into separate columns.
- **JSON**: Writes a fully parsed JSON representation of the question bank.
- **PDF & Word (DOCX)**: Compiles formatted pages containing test papers and answer key booklets.
